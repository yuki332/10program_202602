# Phase 1: CLIプログラムでIPOを身につける

## 目的

**CLIのシンプルなプログラムを作りながら、IPO（Input-Process-Output）の考え方を身につける。**

---

## IPOとは？

すべてのプログラムは「何かを受け取り、処理して、結果を返す」もの。これを **IPO** と呼ぶ。

```
Input（入力）  →  Process（処理）  →  Output（出力）
何を受け取る？     何をする？          何を返す？
```

### 日常のIPO例

| 例 | Input | Process | Output |
|----|-------|---------|--------|
| 自動販売機 | お金、ボタン押下 | 金額確認、商品準備 | 商品、お釣り |
| Google検索 | 検索キーワード | DB検索、順位付け | 検索結果一覧 |

---

## IPOカード

コードを書く**前に**、必ずIPOカードを記入する。

| 項目 | 内容 |
|------|------|
| **Input** | |
| **Process** | |
| **Output** | |

**なぜ先に書くか？**
- 作りながら迷子にならない
- AIへの指示が正確になる
- 完成後に設計通りか確認できる

## 作業の流れ

各Milestoneで以下の流れを繰り返す：

```
1. ブランチ作成  →  2. IPOカード記入  →  3. コード実装  →  4. Push & PR作成
```

## Milestone 1: おしゃべり挨拶ボット

### 課題

名前を入力すると、時間に応じた挨拶を返すCLIツールを作る。

### 進め方

1. **ブランチ作成**
VS Codeでブランチを作成

2. **IPOカードを記入**
`ipo_card.md` として保存

3. **AIに指示してコード生成**
`greeting.js` を作成、I/P/O コメントを追加

4. **Push & PR作成**
VS Codeで該当ファイルをAdd, Commit, Pushする
GitHubでPull Requestを作成

### 完了条件
- [ ] IPOカード（`ipo_card.md`）をコミットした
- [ ] 3パターンの挨拶が動作する
- [ ] コードに I/P/O コメントがある
- [ ] Pull Requestを作成した

## Milestone 3: ToDoリスト

### 課題

タスクを追加できるCLIツールを作る。

### 進め方

1. **ブランチ作成**
VS Codeでブランチを作成

2. **IPOカードを3つ記入**
`ipo_card.md` として保存（機能ごとにIPOがある）

3. **AIに指示してコード生成**

4. **Push & PR作成**
VS Codeで該当ファイルをAdd, Commit, Pushする
GitHubでPull Requestを作成

### 注意: データの揮発性

このToDoリストは、プログラム終了時にデータが消える。
これは意図的な設計。Phase 2でデータベースによる永続化を学ぶ。

### 完了条件
