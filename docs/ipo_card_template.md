# IPOカード テンプレート

## IPOとは？

**IPO = Input - Process - Output**

全ての機能・処理は、この3つの要素で説明できます。
コードを書く（AIに指示する）前に、必ずこのカードを記入してください。

---

## IPOカード

### 機能名: ＿＿＿＿＿＿＿＿＿＿

| 項目 | 内容 |
|------|------|
| **Input (きっかけ)** | ユーザーが何を入力し、何をするか？ |
| **Process (裏側の処理)** | サーバーは何を受け取り、DBに何を保存/削除するか？ |
| **Output (結果)** | 処理後、画面はどう変化し、データはどうなっているか？ |

---

## 記入例

### 機能名: タスクの追加

| 項目 | 内容 |
|------|------|
| **Input** | ユーザーがテキストボックスに「買い物に行く」と入力し、「追加」ボタンをクリック |
| **Process** | サーバーがタスク名を受け取り、データベースのtasksテーブルにINSERT |
| **Output** | 画面のタスクリストに「買い物に行く」が追加表示される。DBにはレコードが1件増えている |

## 3層構造の確認

IPOを考える際、**今どこを触っているか**を常に意識してください。

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Client    │ ──► │   Server    │ ──► │  Database   │
│  (ブラウザ)  │ ◄── │  (Node.js)  │ ◄── │  (SQLite)   │
└─────────────┘     └─────────────┘     └─────────────┘
   F12で確認         ターミナルで確認       ファイルに保存
```

## チェックリスト

コードを書く前に、以下を確認してください：

- [ ] Input: 何がきっかけで処理が始まるか明確か？
- [ ] Process: どこで何が行われるか説明できるか？
- [ ] Output: 結果として何が変わるか明確か？
- [ ] 今触っている場所（Client/Server/DB）がわかっているか？
