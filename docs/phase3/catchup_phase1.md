## Day 11: Git基本操作の練習

### 学習目標

- `git branch`, `git checkout`, `git add`, `git commit`, `git push` の5つのコマンドを覚える
- 各コマンドが何をしているかを自分の言葉で説明できる

### 練習1: ブランチの作成と移動

```bash
# 今いるブランチを確認する
git branch

# 練習用ブランチを作成して移動する
git checkout -b practice/day11

# ブランチが変わったことを確認する
git branch
```

**確認ポイント**: `git branch` の出力で、`*` がついているブランチが現在のブランチ。

### 練習2: ファイルの作成とコミット

```bash
# 練習用のファイルを作成する（VS Codeで作成してもOK）
echo "# 練習メモ" > practice_memo.md

# 変更状況を確認する
git status

# ファイルをステージングに追加する
git add practice_memo.md

# 変更状況をもう一度確認する（色が変わるのを観察）
git status

# コミットする
git commit -m "feat: 練習メモファイルを追加"
```

### 練習3: プッシュ

```bash
# GitHubに送信する
git push -u origin practice/day11
```

**確認**: GitHubのリポジトリページを開き、ブランチが表示されることを確認する。

### Git基本コマンド チートシート

| コマンド | 意味 | 例 |
|---------|------|-----|
| `git branch` | ブランチ一覧を表示 | `git branch` |
| `git checkout -b [名前]` | ブランチを作って移動 | `git checkout -b feature/hello` |
| `git checkout [名前]` | ブランチを移動 | `git checkout main` |
| `git status` | 変更状況を確認 | `git status` |
| `git add [ファイル]` | ステージングに追加 | `git add greeting.js` |
| `git commit -m "[メッセージ]"` | 変更を記録 | `git commit -m "feat: 挨拶機能を追加"` |
| `git push` | GitHubに送信 | `git push` |
| `git pull` | GitHubから取得 | `git pull` |

### 実験: git status の変化を観察する

1. `git status` を実行する（何も表示されない = クリーンな状態）
2. ファイルを1つ作成する
3. `git status` を実行する（赤い文字で表示される = 未追跡ファイル）
4. `git add` する
5. `git status` を実行する（緑の文字で表示される = ステージング済み）
6. `git commit` する
7. `git status` を実行する（再びクリーンな状態）

**実験レポート**:

```
git status の変化
  ファイル作成前の表示:（実験後に書く）
  ファイル作成後の表示:（実験後に書く）
  git add 後の表示:（実験後に書く）
  git commit 後の表示:（実験後に書く）
  わかったこと:（赤と緑の違いは何か）
```

## Day 12: Milestone 1 に取り組む

### 学習目標

- IPOカードを作成して、プログラムの設計を理解する
- AIにIPOカードを渡してコードを生成させる
- 生成されたコードを実行して動作確認する

### ステップ 1: ブランチを作成する

```bash
git checkout main
git pull
git checkout -b feature/milestone1-greeting
```

### ステップ 2: IPOカードを作成する

`ipo_card.md` として以下のテンプレートを埋める。

```markdown
# Milestone 1: おしゃべり挨拶ボット

## IPOカード

| 項目 | 内容 |
|------|------|
| **Input** | ユーザーが名前を入力する |
| **Process** | 現在の時間を取得し、時間帯に応じた挨拶を選ぶ |
| **Output** | 「おはようございます、[名前]さん！」のように挨拶を表示する |

## 仕様

- 朝（5:00-11:59）: 「おはようございます」
- 昼（12:00-17:59）: 「こんにちは」
- 夜（18:00-4:59）: 「こんばんは」
```

### ステップ 3: AIにコードを生成させる

IPOカードの内容をAI（Copilot や ChatGPT）に渡して、`greeting.js` を生成させる。

**AIへの指示例**:

```
以下のIPOカードに基づいて、Node.jsのCLIプログラムを作ってください。

- Input: ユーザーがコマンドライン引数で名前を入力する
- Process: 現在の時間を取得し、時間帯に応じた挨拶を選ぶ
  - 朝（5:00-11:59）: 「おはようございます」
  - 昼（12:00-17:59）: 「こんにちは」
  - 夜（18:00-4:59）: 「こんばんは」
- Output: 「おはようございます、[名前]さん！」のように表示する

ファイル名: greeting.js
実行方法: node greeting.js [名前]
```

### ステップ 4: コードを実行する

```bash
node greeting.js 田中
# おはようございます、田中さん！（朝の場合）
```

**テスト項目**:

- [ ] 名前を入れて実行 → 挨拶が表示される
- [ ] 名前を入れないで実行 → エラーにならないか確認
- [ ] 時間帯が正しく判定されるか確認

### ステップ 5: コミットしてプッシュする

```bash
git add ipo_card.md greeting.js
git commit -m "feat: Milestone 1 おしゃべり挨拶ボットを作成"
git push -u origin feature/milestone1-greeting
```

### ステップ 6: PRを作成する

GitHubでPRを作成する。

```
タイトル: feat: Milestone 1 おしゃべり挨拶ボットを作成

本文:
## 変更内容
- IPOカード（ipo_card.md）を作成
- 挨拶ボット（greeting.js）を作成

## テスト方法
1. `node greeting.js 田中` で実行
2. 時間帯に応じた挨拶が表示されることを確認
```

## Day 13: 動作確認とMilestone 1 完了

### 学習目標

- さまざまな入力パターンでテストする習慣を身につける
- コードを修正してコミットする流れを体験する

### ステップ 1: テスト項目を確認する

以下の入力パターンで動作を確認する:

| テスト | 入力 | 期待する結果 |
|-------|------|------------|
| 通常 | `node greeting.js 田中` | 時間帯に応じた挨拶 |
| 名前なし | `node greeting.js` | エラーにならない |
| スペース付き | `node greeting.js "山田 太郎"` | 正しく挨拶される |

### ステップ 2: 問題があれば修正する

テストで問題が見つかったら、修正して再度コミットする。

```bash
# ファイルを修正した後
git add greeting.js
git commit -m "fix: 名前なしの場合のエラーハンドリングを追加"
git push
```

### ステップ 3: PRをマージする

講師にレビューを依頼し、Approve されたらマージする。

```bash
# マージ後、mainを更新
git checkout main
git pull
```

## 困ったときは

### よくあるトラブル

| 問題 | 原因 | 解決方法 |
|-----|------|---------|
| `git push` でエラーが出る | ブランチが紐づいていない | `git push -u origin [ブランチ名]` を使う |
| `git checkout` でエラーが出る | コミットしていない変更がある | 先に `git add` → `git commit` する |
| ファイルが見つからない | 違うブランチにいる | `git branch` で現在のブランチを確認 |
| AIが生成したコードが動かない | 仕様が不明確 | IPOカードを見直して、AIに再度指示する |

### 質問の仕方

質問するときは、以下の3つを伝えると解決が早い:

1. **何をしようとしたか**（「ブランチを作ろうとした」）
2. **何が起きたか**（「エラーメッセージが出た」→ メッセージをコピペ）
3. **何を試したか**（「ブランチ名を変えてみたが同じエラーだった」）
